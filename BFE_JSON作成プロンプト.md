## BusinessFlowEditor とは

BusinessFlowEditor（以下 BFE）は、社内業務フローを **スイムレーン図** として可視化するブラウザ上の HTML 単体ツールです。

画面構成：
- **レーン（列）**：業務の担当組織・部署ごとの縦列（例: 資材課 ／ 倉庫 ／ 仕入先）
- **グループ（横帯）**：業務フェーズ・大区分ごとの横帯（例: 見積・調達 ／ 入荷・検収）
- **ノード（四角形）**：レーンとグループの交点に置く個別の業務ステップ（例: 発注 ／ 検収）
- **エッジ（矢印）**：ノード間の情報・モノの流れ

BFE は JSON ファイルを読み込んでこの図を描画します。
今回の作業は「ドキュメントを読んでこの JSON を生成する」ことです。

## 作業手順

JSON を生成する前に、次の順序でドキュメントを分析してください。

### STEP 1: 担当組織の抽出 → lanes 設計

ドキュメントに登場する担当者・部署・システム・外部組織を列挙し、
スイムレーン（列）として設計します。

- 左から右へ「外部 → 社内（依頼側） → 社内（実務側） → 倉庫・現場」の順が読みやすい
- 1 フローで 3〜7 列が適切
- システム名のみの列（MAPS, GRANDIT等）は「システム列」として追加してもよい

### STEP 2: 業務フェーズの抽出 → groups 設計

ドキュメントの章・節・大見出しを「業務フェーズ（グループ）」にマッピングします。

- 配列の先頭 = 図の上端、末尾 = 図の下端
- 「〜処理」「〜確認」「〜登録」等の動詞句がグループ名の候補
- 1 フローで 2〜5 グループが適切（大きな業務フェーズ単位）

### STEP 3: 業務ステップの抽出 → nodes 設計

各フェーズの手順ステップをノードにマッピングします。

- 1 ノード = 1 担当者（組織）が行う 1 つの **業務プロセス**
- ラベルは 12 文字以内の動詞句で簡潔に（長い説明は note フィールドへ）
- 同じフェーズで同時並行で行われるステップは **同じ y 値** にする（横並び表示）
- 手順の前後は **異なる y 値** にする（上が先、下が後）

#### 粒度ガイドライン（重要）

ノードの粒度は「業務全体の俯瞰図」レベルに合わせること。

**適切な粒度（1ノード = 1業務プロセス）:**
- 「見積」「発注」「廃却部品選定」「承認」「棚入れ」
- 1フローあたり **6〜12 ノード** が目安

**細かすぎる粒度（NG）:**
- 「MAPSメニューを開く」「ファイル出力ボタン押下」「PDFを添付」
- 「月報データ取得」→「前年度データ取得」→「データ統合」のような
  1つの業務プロセス内の個別操作をノードに分割しない

**判断基準:**
- 担当者（組織）が異なる → 別ノード
- 同じ担当者の連続操作 → 1ノードにまとめ、詳細は note に記載
- MAPS等のシステム操作 → ノードではなく system フィールドや note で表現

### STEP 4: 情報の流れの抽出 → edges 設計

ノード間の「渡す」「受け取る」「通知する」「照合する」等を矢印で表現します。

### STEP 5: マニュアル番号の割当 → manual / manualStatus 設定

ドキュメントに PSM-XX000 形式のマニュアル番号が記載されていれば各ノードに設定します。
記載がなければ全ノードで `"manual": "", "manualStatus": ""` とします。

## JSON 仕様

### トップレベル構造

```json
{
  "meta":    { },
  "lanes":   [ ],
  "groups":  [ ],
  "nodes":   [ ],
  "edges":   [ ],
  "manuals": [ ]
}
```

---

### meta

| フィールド | 型 | 説明 |
|---|---|---|
| title | string | フロー図のタイトル（日本語可） |
| version | string | 固定値 `"1.0"` |
| lastUpdated | string | 今日の日付 `"YYYY-MM-DD"` |

---

### lanes（スイムレーン・縦列）

```json
{
  "id":    "英数字とアンダースコアのみ（例: shizai, dept_a）",
  "label": "表示名（日本語可、例: 資材課）",
  "color": "淡いパステル系 CSSカラー（後述のパレットから選択）",
  "width": 120 〜 350 の整数
}
```

**制約:**
- `id` はこのファイル内で一意。日本語・スペース・記号は使用不可。
- `lanes` の配列順 = 左から右へ表示される列順。
- `width` の目安: `最長ラベル文字数 × 14 + 40`（例: 8文字 → 152）

**推奨カラーパレット（淡いパステル）:**
```
#E8F4FD  薄い水色（社内部署の標準）
#E8F5E9  薄い緑（現場・倉庫系）
#B0C4DE  水色グレー（外部・仕入先）
#B2EBF2  薄いシアン（サービス・営業）
#FFF9C4  薄い黄（経理・管理系）
#F3E5F5  薄い紫（IT・システム系）
#FFE0B2  薄いオレンジ（承認・決裁系）
```

---

### groups（業務フェーズ・横帯）

```json
{
  "id":    "英数字とアンダースコアのみ（例: receiving, approval）",
  "label": "表示名（日本語可、例: 入荷・検収）",
  "color": "#HEX（省略可。設定する場合は極薄い色、省略時は白）"
}
```

**制約:**
- `id` はこのファイル内で一意。
- `groups` の配列順 = 上から下へ表示されるフェーズ順。

---

### nodes（業務ステップ）

```json
{
  "id":           "n + 連番整数（例: n01, n02 … n100）。全ノードで一意。",
  "label":        "ノードラベル（12文字以内の動詞句）",
  "lane":         "lanes[].id のいずれか（存在しない値は不可）",
  "group":        "groups[].id のいずれか（存在しない値は不可）",
  "x":            0（通常） または 正の整数（同レーン内で横にずらす場合のみ）,
  "y":            整数（後述の y 値ルールに従う）,
  "manual":       "" または "PSM-XX000 形式のマニュアルID",
  "manualStatus": "" または "完了" または "作成中" または "未着手",
  "system":       "" または "関連システム名（例: MAPS）",
  "note":         "" または "補足テキスト（改行は \\n）"
}
```

#### y 値ルール（最重要）

y 値はキャンバス上の絶対縦座標（ピクセル）です。
以下のルールに従って整数値を割り当ててください。

```
ルール1: 同じグループ内で「同時・並行」するステップ → 同じ y 値
         （例: 資材課が「発注」する時、仕入先が「受注」する = 同じ y）

ルール2: 同じグループ内で「前後関係」があるステップ → y を 40〜50 増やす

ルール3: グループが変わるとき → 前グループの最大 y + 80〜120 から開始

ルール4: 最初のグループの最初のステップ → y = 40 から開始

ルール5: y 値は整数のみ。小数点禁止。
```

**y 値の割り当て例（3グループ・複数ステップの場合）:**

```
グループ A「見積・調達」
  ステップ1 (y=40):  資材課:見積依頼, 仕入先:見積依頼受信
  ステップ2 (y=80):  仕入先:見積書送付, 資材課:見積書受取
  ステップ3 (y=120): 資材課:見積承認

グループ B「発注」  ← 120 + 100 = 220 から開始
  ステップ1 (y=220): 資材課:発注, 仕入先:受注
  ステップ2 (y=260): 資材課:発注確認

グループ C「入荷・検収」  ← 260 + 100 = 360 から開始
  ステップ1 (y=360): 倉庫:受却
  ステップ2 (y=400): 倉庫:入荷・検品
  ステップ3 (y=440): 資材課:原入処理, 倉庫:原入処理
```

#### x 値ルール

- 通常は `0` を使用。
- 同じ `lane` + 同じ `group` に y 値が近い複数ノードを置く場合のみ、
  2 番目以降に `x: 150` 等を設定して横にずらす。

#### manual / manualStatus フィールドの意味

これらは「この業務ステップに対応する業務マニュアル」を紐づけるためのフィールドです。

- `manual`: マニュアル番号（例: `"PSM-WH001"`）。形式は `PSM-[大文字2〜4文字][3桁数字]`。
- `manualStatus`: そのマニュアルの作成進捗。
  - `"完了"` = マニュアル作成済み・承認済み
  - `"作成中"` = 作成中またはレビュー中
  - `"未着手"` = 未着手
  - `""` = マニュアル対象外または番号不明

**対応ルール:**
- `manual` を設定した場合は必ず `manualStatus` も `""` 以外を設定する
- `manual` を設定した場合は `manuals` 配列にも同じ `id` を追加する
- ドキュメントにマニュアル番号の記載がなければ全ノードで両方 `""` でよい

---

### edges（矢印・接続）

```json
{
  "from":  "nodes[].id（矢印の出発点）",
  "to":    "nodes[].id（矢印の到着点）",
  "label": "" または "矢印に付けるラベル文字列"
}
```

**制約:**
- `from` と `to` は必ず `nodes` 配列内に存在する `id` を参照する
- 同じ `(from, to)` の組み合わせは 1 件のみ（重複禁止）
- `from === to` の自己ループ禁止
- `label` は重要な情報名・書類名・条件のみ記載。不要なら `""`

---

### manuals（省略可・マニュアルマスタ）

`nodes` に `manual` フィールドを設定した場合のみ追加します。

```json
{
  "id":     "マニュアルID（nodes[].manual と一致させる）",
  "status": "完了" または "作成中" または "未着手"
}
```

`nodes[].manualStatus` と `manuals[].status` の値は一致させてください。
マニュアルへの言及がないドキュメントでは `"manuals": []` とします。

---

## 出力前の自己チェックリスト（必須）

出力する前に以下を確認し、問題があれば修正してから出力してください。

```
構文
  □ JSON として構文エラーがない
  □ 末尾カンマが存在しない
  □ 全ての文字列値がダブルクォートで囲まれている
  □ 数値フィールド（x, y, width）に引用符がない

一意性
  □ lanes[].id が全て異なる
  □ groups[].id が全て異なる
  □ nodes[].id が全て異なる

参照整合性
  □ 全 nodes[].lane が lanes[].id に存在する
  □ 全 nodes[].group が groups[].id に存在する
  □ 全 edges[].from が nodes[].id に存在する
  □ 全 edges[].to が nodes[].id に存在する
  □ edges に (from, to) の重複がない

manuals 整合性（manuals を使う場合のみ）
  □ nodes[].manual で参照している全 ID が manuals[].id に存在する
  □ nodes[].manualStatus と対応する manuals[].status が一致している

y 値
  □ 各グループの先頭ノードの y が前グループの最大 y + 80 以上
  □ y に小数点が含まれていない

フィールド値
  □ manualStatus の値が "" / "完了" / "作成中" / "未着手" のいずれかのみ
  □ lanes[].width が 120〜350 の範囲内
```

## 出力形式

以下の順序で出力してください。

### 1. 設計メモ（日本語・箇条書き）

- レーン構成: なぜこの列分けにしたか
- グループ構成: なぜこのフェーズ分けにしたか
- ノード数・エッジ数の概要
- manuals 設定の有無と理由
- ドキュメントから読み取れなかった情報（不明点）

### 2. JSON（インデント付き・整形済み）

````json
{
  ...
}
````

## 絶対に守る禁止事項

- `lanes[].id`, `groups[].id`, `nodes[].id` に **日本語・スペース・ハイフン・ドット** を含めない
- `nodes[].label` は **13文字以上** にしない（超える場合は `note` に移す）
- 存在しない `id` を `lane`, `group`, `from`, `to` で参照しない
- JSON コードブロックの **外側** に波括弧・角括弧・カンマ等の JSON 断片を含めない
- `manualStatus` に `"完了"` / `"作成中"` / `"未着手"` / `""` 以外の文字列を使わない
- `y` 値に小数点を使わない（整数のみ）
- JSON の中に `// コメント` を書かない（JSON はコメント非対応）

## トラブルシューティング

| 症状 | 原因の可能性 | 対処 |
|------|------------|------|
| 読込後に何も表示されない | `nodes` が空、または `lane`/`group` の参照が壊れている | 参照整合性チェックリストを再確認 |
| ノードが縦に重なって表示される | 同グループ内の異なるステップで y が同じ | 各ステップの y を 40〜50 ずつ変える |
| グループをまたいでノードが重なる | グループ間の y ジャンプが小さすぎる | グループ切り替え時に +80 以上ジャンプ |
| JSON 読込エラーが出る | 末尾カンマ・シングルクォート・コメント等の構文エラー | 「JSON構文エラーを修正して」と追加依頼 |
| 矢印が表示されない | `from`/`to` が存在しない node.id を参照している | 参照整合性チェックリストを再確認 |
| マニュアルバッジが表示されない | `manual` フィールドが設定されているが `manuals` 配列に対応 id がない | `manuals` 配列に id を追加する |
